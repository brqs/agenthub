# AgentHub 前端发布交接说明

> 用途：用于 B2 部署发布能力与前端开发人员对接。  
> 最后核对：2026-06-01

## 1. 当前结论

当前远端前端仍需要更新构建产物。

- 线上访问地址：`http://154.44.25.94:1573`
- 后端公网地址：`http://111.229.151.159:8000`
- 前端由 Nginx 托管静态产物，并将同源 `/api/` 请求反向代理到后端。
- 本地前端已包含基础版 `deployment_status` 部署状态卡。
- 远端静态资源 hash 与本地 `dist/` 不一致，远端 bundle 未包含“打开部署地址”“下载源码”等状态卡专属文案，因此线上仍是旧构建。
- 当前仓库没有未提交的前端代码改动；需要发布的是已经存在于仓库中的最新前端代码。

基础版部署状态卡已经支持：

- 展示 `publishing`、`published`、`failed`、`stopped`、`not_supported` 状态。
- 打开静态站点部署 URL。
- 下载源码 zip。
- 展示失败原因与简短日志。

## 2. 当前 MVP 的发布方式

如需先验证现有 Deployment / Release MVP，前端开发人员拉取最新代码后执行：

```bash
cd frontend
pnpm install
pnpm build
```

然后将 `frontend/dist/` 发布到 `http://154.44.25.94:1573` 对应的 Nginx 静态目录。

发布完成后至少验证：

```bash
curl -I http://154.44.25.94:1573
curl -I http://154.44.25.94:1573/api/v1/agents
```

浏览器侧需要确认：

- AgentHub 首页能够正常打开。
- 登录、聊天和 SSE 流式消息不受影响。
- 聊天中出现 `deployment_status` block 时，状态卡能够正常渲染。
- 静态站点 URL 可以新窗口打开。
- 源码包下载按钮可以正常下载 zip。
- 容器化部署暂不支持时，卡片显示 `Not supported`，而不是前端报错。

## 3. 第五点部署发布完善后的统一上线

课程设计第五点要求：

- 聊天中发送“部署”指令后返回部署状态卡片。
- 一键生成预览 URL。
- 支持静态站点部署。
- 支持容器化部署。
- 支持源码打包下载。

当前基础 MVP 已经覆盖状态卡、静态 URL、源码 zip 和容器化 `not_supported` 占位。下一阶段会继续完善：

- 静态发布与临时 Preview 生命周期解耦。
- 使用不可变 snapshot 和公开只读 Token URL。
- 增加停止发布按钮。
- 增加发布状态自动刷新。
- 增加复制 URL 操作。
- 增加发布历史列表。
- 增加源码包过期与清理提示。
- 增加容器发布安全底座；本阶段不直接开放 Docker 或宿主机 shell。

为了避免重复上线，建议在上述前后端增强合并后统一发布一次前端。

## 4. 需要前端开发人员提供的信息

仓库目前没有记录远端前端的正式发布入口。本机也无法使用现有 SSH key 登录 `ubuntu@154.44.25.94`。

请前端开发人员提供以下任意一种发布方式：

1. 现有 CI/CD 发布命令或操作入口。
2. 可登录服务器的 SSH 用户、临时公钥授权方式，以及 Nginx 静态目录。
3. 由前端开发人员负责发布：后端增强合并后，由其拉取对应 PR 分支并发布 `dist/`。

还需要确认：

- Nginx 静态目录是否为 `/usr/share/nginx/html` 或其他路径。
- 发布后是否需要 reload Nginx。
- 前端是否由 Docker、systemd 或其他方式管理。
- 是否存在测试环境，避免直接覆盖正式环境。

不要通过聊天发送密码或 SSH 私钥。需要授权时，只提供临时账号或 SSH 公钥授权。

## 6. 联调验收边界

前端发布入口确认后，部署发布能力的完整联调按以下顺序执行：

1. 后端完成 migration、重启和健康检查。
2. 前端完成 `pnpm build` 与远端静态产物发布。
3. 从 `http://154.44.25.94:1573` 登录真实账号。
4. 通过群聊向 Orchestrator 发送预览、静态发布、源码打包和容器化部署命令。
5. 验证状态卡、公开 URL、源码 zip、停止发布和 `not_supported` 状态。
6. 保留 SSE、workspace、deployment record 和浏览器验收报告。
